# 附录B：研究设计检查清单

本附录为SIO-MMOS丛书实践指南提供系统化的研究设计检查清单，涵盖研究规划、数据收集和分析方法三个核心阶段。检查清单旨在帮助研究者确保研究设计的严谨性、科学性和可重复性。建议在使用本清单时，针对具体研究类型（定量研究、定性研究、混合方法研究或计算研究）灵活调整检查项的应用范围和重点。

## B.1 研究规划检查清单

研究规划阶段是整个研究项目的基础，良好的规划能够提高研究效率、降低风险、确保研究质量。本部分检查清单涵盖研究问题定义、理论基础、研究设计选择、资源规划和伦理考虑五个关键维度。

### B.1.1 研究问题定义检查项

研究问题是整个研究项目的核心导向，明确、恰当的研究问题对于研究成功至关重要。

- [ ] **研究问题是否明确陈述了研究的核心问题**
  - 是否使用清晰、无歧义的语言表述研究问题？
  - 研究问题是否聚焦于单一主题或有限的一组相关问题？
  - 是否避免了过于宽泛或过于狭窄的问题界定？
  - 研究问题的表述是否与研究目标和假设保持一致？

- [ ] **研究问题是否具有可研究性（可通过经验或理论方法回答）**
  - 研究问题是否可以通过实证数据收集和分析来回答？
  - 是否可以通过理论分析、文献综述或概念分析来探讨？
  - 研究问题是否避免了价值判断或规范性陈述的过度影响？
  - 是否存在可行的研究方法和技术路径来回答该问题？

- [ ] **研究问题是否具有学术或实践意义**
  - 研究问题是否填补了现有文献的知识空白？
  - 是否对理论发展、概念澄清或模型构建有贡献？
  - 研究问题是否具有实践应用价值或政策相关性？
  - 是否对解决现实世界问题或改进实践有潜在影响？

- [ ] **研究问题是否界定清晰，范围适中**
  - 研究问题的边界是否明确划定？
  - 是否明确了研究的时间、空间、对象和变量范围？
  - 研究范围是否与可用资源（时间、资金、人力）相匹配？
  - 是否避免了范围蔓延或过度承诺的风险？

- [ ] **研究问题是否与现有文献存在明确对话关系**
  - 研究问题是否基于对现有文献的系统综述？
  - 是否明确指出了与现有研究的联系、分歧或扩展？
  - 研究问题是否回应了文献中识别出的理论缺口或实证矛盾？
  - 是否建立了清晰的理论对话框架或研究对话脉络？

### B.1.2 理论基础检查项

理论基础为研究提供概念框架和分析视角，是研究严谨性的重要保证。

- [ ] **是否完成了系统的文献综述**
  - 是否进行了全面的文献检索，覆盖主要数据库和灰色文献？
  - 文献综述是否遵循系统化和可重复的方法？
  - 是否对检索到的文献进行了质量评估和相关性筛选？
  - 文献综述是否涵盖了理论基础、实证研究和方法论文献？

- [ ] **是否识别了核心理论框架**
  - 是否明确识别了支撑研究的核心理论或理论流派？
  - 理论框架的选择是否有充分的理论依据？
  - 是否考虑了替代理论视角和竞争性解释？
  - 理论框架是否与研究问题和研究设计相匹配？

- [ ] **关键概念是否已明确界定**
  - 研究中的核心概念是否有清晰的操作性定义？
  - 概念界定是否基于文献回顾和理论分析？
  - 是否区分了概念的定义性特征和操作性特征？
  - 概念界定是否避免了歧义和循环定义？

- [ ] **概念间关系是否已澄清**
  - 核心概念之间的逻辑关系是否已明确？
  - 是否识别了自变量、因变量、调节变量和中介变量？
  - 概念关系的方向性、强度和条件是否已明确？
  - 是否建立了概念关系图或理论模型？

- [ ] **理论假设是否已明确陈述**
  - 基于理论框架推导出的研究假设是否已明确陈述？
  - 假设是否具有可检验性（可通过实证数据验证或证伪）？
  - 假设是否明确了变量间关系的预期方向和强度？
  - 是否区分了主要假设和探索性假设？

### B.1.3 研究设计选择检查项

研究设计是将研究问题转化为可操作研究方案的核心环节。

- [ ] **研究方法是否与研究问题匹配**
  - 选择的整体研究范式（实证主义、诠释主义、批判理论等）是否恰当？
  - 定量、定性或混合方法的选择是否有充分依据？
  - 研究方法是否能够有效回答研究问题？
  - 是否考虑了研究问题的性质对方法选择的约束？

- [ ] **研究设计类型选择是否适当**
  - 实验设计、准实验设计、调查设计、民族志、案例研究等选择是否恰当？
  - 设计类型是否与研究目的（探索、描述、解释、预测）相匹配？
  - 是否考虑了内部效度、外部效度、构念效度和统计结论效度的威胁？
  - 是否制定了应对效度威胁的策略？

- [ ] **时间维度考虑是否周全**
  - 横断面设计、纵向设计或队列设计的选择是否有充分依据？
  - 研究时间跨度是否足以捕捉感兴趣的动态过程或变化？
  - 测量时间点或观察时间点的选择是否有理论依据？
  - 是否考虑了时间效度（历史事件、成熟效应等）的威胁？

- [ ] **可行性是否已评估**
  - 研究设计在现有时间、资源和技术条件下是否可行？
  - 是否评估了数据收集、分析和报告的实际可操作性？
  - 是否识别了潜在的执行障碍并制定了应对策略？
  - 是否考虑了研究者自身的能力和经验局限？

- [ ] **潜在局限是否已识别并计划应对**
  - 研究设计固有的局限性是否已被识别和记录？
  - 是否制定了减轻或应对这些局限的策略？
  - 研究结论的适用范围和边界是否已明确？
  - 是否考虑了替代解释和竞争性假设的检验？

### B.1.4 资源规划检查项

资源规划确保研究项目有充足的物质和人力资源支持。

- [ ] **详细的时间线是否已制定**
  - 是否建立了详细的项目时间线，包括所有关键任务和依赖关系？
  - 是否识别了关键里程碑和阶段性目标？
  - 是否为每个阶段分配了合理的时间预算？
  - 是否制定了时间缓冲和应急计划以应对延误？
  - 时间线是否考虑了伦理审批、数据收集窗口等外部时间约束？

- [ ] **人力资源是否已规划**
  - 研究团队组成是否合理，技能互补？
  - 每个团队成员的角色和责任是否已明确界定？
  - 是否评估了团队成员的能力和培训需求？
  - 是否制定了沟通协调机制和决策流程？
  - 是否考虑了外部合作者、顾问和利益相关者的参与？

- [ ] **资金预算是否已编制**
  - 是否编制了详细的项目预算，包括所有预期支出？
  - 预算类别是否全面（人员、设备、材料、差旅、间接费用等）？
  - 是否进行了成本估算的准确性评估和不确定性分析？
  - 是否确定了资金来源和申请时间表？
  - 是否制定了财务管理和审计程序？

- [ ] **技术资源是否已识别**
  - 研究所需的硬件资源是否已识别和规划（计算机、服务器、设备）？
  - 所需的软件工具和许可是否已确定和预算？
  - 数据存储、处理和分析的基础设施是否已评估？
  - 网络安全和数据保护措施是否已规划？
  - 技术支持和维护安排是否已建立？

- [ ] **风险预案是否已制定**
  - 是否进行了系统的风险识别和评估？
  - 是否制定了风险应对策略（规避、减轻、转移、接受）？
  - 是否为关键风险制定了应急预案？
  - 是否建立了风险监控和报告机制？
  - 是否购买了适当的保险（如需要）？

### B.1.5 伦理考虑检查项

伦理考量是确保研究符合道德标准、保护参与者权益的关键环节。

- [ ] **知情同意程序是否已设计**
  - 是否制定了清晰、全面的知情同意书？
  - 知情同意书是否使用参与者能理解的语言，避免专业术语？
  - 是否充分披露了研究目的、程序、风险和收益？
  - 是否明确说明了参与是自愿的，可随时退出而不受惩罚？
  - 是否考虑了特殊人群的知情同意程序（未成年人、认知障碍者等）？
  - 是否设计了知情同意的记录和存储程序？

- [ ] **隐私保护措施是否已规划**
  - 是否识别了研究过程中收集的个人敏感信息？
  - 是否制定了数据最小化原则，仅收集必要信息？
  - 是否设计了匿名化或假名化方案？
  - 是否制定了数据访问控制策略？
  - 是否考虑了数据跨境传输的合规要求？
  - 是否制定了隐私影响评估程序？

- [ ] **潜在风险是否已评估**
  - 是否识别了研究可能带来的身体风险（如医疗研究中的副作用）？
  - 是否评估了心理风险（如尴尬、焦虑、创伤反应）？
  - 是否考虑了社会风险（如污名化、关系损害）？
  - 是否评估了经济风险（如收入损失、就业机会影响）？
  - 是否考虑了法律风险（如自曝违法行为的潜在后果）？
  - 是否评估了风险与收益的平衡？

- [ ] **风险最小化策略是否已制定**
  - 是否设计了排除高风险参与者的筛选标准？
  - 是否制定了风险监测和早期退出机制？
  - 是否准备了危机干预资源（心理咨询、医疗支持）？
  - 是否设计了数据安全保障措施以防止数据泄露？
  - 是否制定了意外事件的报告和应对程序？
  - 是否确保风险最小化措施不会影响研究效度？

- [ ] **伦理审批流程是否已启动**
  - 是否识别了适用的伦理审查委员会（IRB/REC）？
  - 是否了解了伦理审批的要求和程序？
  - 是否准备了完整的伦理审查申请材料？
  - 是否预留了充足的伦理审批时间？
  - 是否制定了应对伦理审查意见的修改计划？
  - 是否考虑了多中心研究或多国研究的伦理协调？
  - 是否建立了持续伦理监督机制？

- [ ] **数据管理和共享计划是否已制定**
  - 是否制定了数据管理计划（DMP），涵盖数据收集、存储、处理和共享？
  - 是否确定了数据共享的策略和范围（完全开放、受控访问、不共享）？
  - 是否选择了适当的数据存储库或平台？
  - 是否制定了数据格式和元数据标准？
  - 是否考虑了数据长期保存和可访问性？
  - 是否制定了知识产权和作者归属政策？
  - 是否符合资助机构、期刊和机构的数据政策要求？

## B.2 数据收集检查清单

数据收集是研究过程中将研究设计转化为实际数据的关键环节。数据质量直接影响研究结论的可靠性和有效性。本部分检查清单涵盖数据收集方法、数据质量管理、数据安全和元数据记录四个核心维度。

### B.2.1 数据收集方法检查项

数据收集方法的选择和实施直接决定了数据的效度和信度。

- [ ] **数据收集方法是否与研究问题匹配**
  - 所选数据收集方法（问卷调查、访谈、观察、实验、文献分析、大数据抓取等）是否能够捕捉研究问题所需的信息？
  - 方法的选择是否有充分的理论和方法论依据？
  - 是否考虑了研究问题对不同数据类型（定量/定性/混合）的需求？
  - 数据收集方法是否与理论框架和研究假设相匹配？
  - 是否评估了替代数据收集方法的优劣？

- [ ] **测量工具信度是否已评估**
  - **重测信度**：是否在适当的时间间隔内对同一组被试进行了两次测量，并计算了两次得分的一致性？
  - **内部一致性信度**：对于多项目量表，是否计算了Cronbach's α系数或McDonald's ω系数，且值是否达到可接受水平（通常α≥0.70）？
  - **评分者间信度**：对于需要主观判断的测量，是否评估了不同评分者之间的一致性（如Kappa系数、ICC组内相关系数）？
  - **分半信度**：是否将测量工具分为等价的两半，并计算了两半得分的一致性？
  - 信度评估的样本量是否足够（通常建议至少100-200人）？

- [ ] **测量工具效度是否已评估**
  - **内容效度**：测量工具是否全面覆盖了所要测量的构念领域？是否邀请了专家评估项目的代表性？
  - **构念效度**：
    - **聚合效度**：测量同一构念的不同工具或方法是否产生高度相关的结果？
    - **区分效度**：测量不同构念的工具是否产生低相关的结果？
    - 是否进行了探索性因子分析（EFA）或验证性因子分析（CFA）来评估结构效度？
  - **效标效度**：
    - **同时效度**：测量结果与当前效标（已确立的标准测量）的相关性是否足够高？
    - **预测效度**：测量结果对未来行为或结果的预测能力如何？
  - 效度评估是否基于充分的理论依据和实证证据？

- [ ] **抽样策略是否已明确**
  - **概率抽样**：
    - **简单随机抽样**：是否每个研究单元都有已知的、非零的被选概率？
    - **分层随机抽样**：是否根据关键特征将总体分层，并在各层内进行随机抽样？
    - **整群抽样**：是否将总体分为群组，随机选择群组并对选中群组的所有成员或子样本进行研究？
    - **系统抽样**：是否按照固定的间隔从有序总体中选择样本？
  - **非概率抽样**：
    - **便利抽样**：是否基于可得性和便利性选择样本？这种选择的局限性是否已被认识？
    - **判断抽样（目的抽样）**：是否基于研究者的判断选择具有特定特征的信息丰富的案例？
    - **滚雪球抽样**：是否通过已有参与者推荐新参与者，适用于难以接触的群体？
    - **配额抽样**：是否根据预设的比例在各类别中选择便利样本？
  - **样本量**：
    - 样本量是否基于统计功效分析确定？
    - 对于定量研究，是否考虑了效应量、显著性水平（α）和统计功效（1-β）？
    - 对于定性研究，是否考虑了信息饱和原则和理论抽样的需要？
    - 是否考虑了预期的缺失数据率和失访率？

- [ ] **数据收集程序是否已标准化**
  - **培训**：
    - 数据收集人员是否接受了充分的培训？
    - 培训内容是否涵盖了数据收集工具的使用、标准操作程序、伦理要求和应急处理？
    - 是否对培训效果进行了评估和认证？
    - 是否建立了持续培训和更新机制？
  - **操作手册**：
    - 是否编写了详细的数据收集操作手册？
    - 手册是否涵盖了数据收集的每个步骤、注意事项和决策规则？
    - 是否包含了示例、模板和常见问题解答？
    - 手册是否易于理解和使用？
  - **质量控制**：
    - 是否建立了数据收集过程中的质量监控机制？
    - 是否进行了定期的数据质量检查和反馈？
    - 是否建立了数据收集偏差的识别和纠正机制？
    - 是否记录了数据收集过程中的所有偏差和异常？

### B.2.2 数据质量管理检查项

数据质量管理确保所收集数据的准确性、完整性、一致性和可靠性。

- [ ] **质量控制措施是否已实施**
  - **监督**：
    - 是否建立了数据收集过程的监督机制？
    - 监督的频率、范围和方式是否适当？
    - 是否使用了现场观察、录音回放、数据抽查等监督手段？
    - 监督结果是否及时反馈给数据收集人员？
  - **检查**：
    - 是否建立了多层次的数据检查机制？
    - 是否进行了实时检查、批次检查和最终检查？
    - 检查的内容是否涵盖数据完整性、逻辑一致性、范围合理性？
    - 是否建立了检查问题的记录和追踪机制？
  - **反馈**：
    - 是否建立了及时、有效的反馈机制？
    - 反馈是否针对具体问题和改进建议？
    - 是否跟踪了反馈后的改进情况？
    - 是否定期总结反馈信息以改进整体流程？

- [ ] **偏差识别与防范机制是否已建立**
  - **选择偏差**：
    - 是否识别了可能影响样本代表性的选择偏差来源？
    - 抽样框架是否完整覆盖了目标总体？
    - 无应答和失访模式是否被监测和分析？
    - 是否使用了统计方法（如倾向得分匹配、加权调整）来减轻选择偏差的影响？
  - **测量偏差**：
    - 测量工具是否存在系统误差？
    - 测量工具的信度和效度是否经过验证？
    - 测量环境是否标准化，以最小化情境变异？
    - 是否考虑了测量工具的文化适应性和语言等效性？
  - **响应偏差**：
    - 是否识别了可能导致响应偏差的来源（社会期望、默许偏差、极端反应、中心趋势）？
    - 问卷设计是否采用了减少响应偏差的策略（随机化、反向计分、间接提问）？
    - 是否使用了验证性问题或方法来检测不诚实或随意回答？
    - 访谈和焦点小组是否建立了促进诚实和开放表达的氛围？
  - **信息偏差**：
    - 回忆偏差是否被考虑，特别是对于过去事件的报告？
    - 信息源（如医疗记录、代理报告）的准确性和完整性是否被评估？
    - 暴露和结局的测量是否相互独立，以避免确认偏差？

- [ ] **数据完整性检查程序是否已设计**
  - **缺失值**：
    - 是否建立了识别和标记缺失数据的标准程序？
    - 是否区分了不同类型的缺失（完全随机缺失MCAR、随机缺失MAR、非随机缺失MNAR）？
    - 是否分析了缺失数据的模式和机制？
    - 是否选择了适当的缺失值处理策略（完整案例分析、多重插补、最大似然估计）？
  - **异常值**：
    - 是否建立了识别异常值的多重方法（统计检验、可视化、领域知识）？
    - 是否区分了极端值、错误值和真正的异常观测？
    - 异常值的处理是否基于对其来源和性质的仔细分析？
    - 是否记录了所有异常值的识别和处理决策？
  - **逻辑一致性**：
    - 是否建立了数据内部的逻辑一致性检查规则？
    - 是否检查了变量之间的逻辑关系（如出生日期早于入学日期）？
    - 是否验证了分类变量的取值范围和数据类型？
    - 是否建立了逻辑错误识别和纠正的工作流程？

- [ ] **数据录入与验证流程是否已规范**
  - **双人录入**：

    - 是否记录了所有数据更正和修改？
  - **自动检查**：
    - 数据录入系统是否内置了范围检查、格式检查和逻辑检查？
    - 是否设置了实时错误提示和阻止机制？
    - 是否建立了数据质量指标和监控仪表板？
    - 是否定期进行数据质量审核和改进？

- [ ] **数据版本控制是否已建立**
  - 是否建立了明确的数据版本命名和标识规则？
  - 是否记录了每个数据版本的变化内容和修改原因？

### B.2.3 数据安全检查项

数据安全保护研究参与者的隐私和数据的安全，是研究伦理和合规的重要组成部分。

- [ ] **存储安全措施是否已实施**
  - **加密**：
    - 静态数据是否使用强加密算法（如AES-256）进行加密？
    - 加密密钥是否安全存储，与数据分离？
    - 密钥管理是否遵循最佳实践（定期轮换、访问控制）？
    - 加密实施是否经过安全测试和验证？
  - **访问控制**：
    - 是否实施了基于角色的访问控制（RBAC）？
    - 是否遵循最小权限原则，仅授予必要的访问权限？
    - 是否建立了用户身份验证机制（强密码、多因素认证）？
    - 是否定期审查和更新访问权限？
  - **防火墙**：
    - 是否在网络安全边界部署了防火墙？
    - 防火墙规则是否经过安全审查和定期更新？
    - 是否监控和记录防火墙日志以检测异常活动？
    - 是否定期进行渗透测试和安全评估？
  - **物理安全**：
    - 数据存储设备是否存放在安全的物理环境中？
    - 是否控制了数据中心的物理访问？

- [ ] **传输安全措施是否已实施**
  - **SSL/TLS**：
    - 所有数据传输是否使用TLS 1.2或更高版本加密？
    - SSL/TLS证书是否来自可信的证书颁发机构？
    - 是否定期更新和轮换证书？
    - 是否禁用了弱加密算法和协议版本？
  - **VPN**：
    - 远程访问是否通过安全的VPN连接？
    - VPN是否使用强加密和认证机制？
    - 是否监控VPN日志以检测异常访问模式？
    - 是否定期审查VPN用户和访问权限？
  - **加密传输**：
    - 文件传输是否使用安全的协议（SFTP、SCP）？
    - 电子邮件中的敏感数据是否加密或使用安全传输方式？
    - API通信是否使用认证和加密机制？
    - 移动设备和可移动媒体的传输是否加密？

- [ ] **访问控制策略是否已制定**
  - **基于角色的访问**：
    - 是否定义了清晰的角色和职责矩阵？
    - 每个角色是否有明确定义的权限范围？
    - 角色分配是否基于工作职责和研究需要？
    - 是否建立了角色变更和撤销的流程？
  - **最小权限原则**：
    - 每个用户是否仅被授予完成其工作所必需的最小权限？
    - 是否定期审查和收紧过度的权限？
    - 特权账户的使用是否受到严格监控和记录？
    - 临时权限是否有明确的到期和撤销机制？
  - **身份认证**：
    - 是否实施了强密码策略（长度、复杂度、更换频率）？
    - 敏感系统和数据是否要求多因素认证（MFA）？
    - 是否使用集中式身份管理系统？
    - 是否定期审查和停用不活跃账户？

- [ ] **备份策略是否已建立**
  - **定期备份**：
    - 是否制定了备份计划，明确备份频率和范围？
    - 关键数据是否进行每日或实时增量备份？
    - 是否定期进行完整备份？
    - 备份是否包括所有关键数据、系统和配置？
  - **异地备份**：
    - 备份副本是否存储在地理位置分离的异地？
    - 是否考虑了自然灾害、火灾、盗窃等风险因素？
    - 云备份服务是否符合数据安全和隐私要求？
    - 异地备份的传输和存储是否加密？
  - **恢复测试**：
    - 是否定期进行备份恢复测试？
    - 恢复测试是否涵盖不同类型的数据和系统？
    - 是否记录了恢复时间目标（RTO）和恢复点目标（RPO）？
    - 是否根据测试结果更新备份策略和程序？
    - 是否制定了灾难恢复计划和应急响应程序？

- [ ] **数据保留与销毁政策是否已制定**
  - **保留期限**：
    - 是否根据法律法规、资助机构要求和研究需要确定了数据保留期限？
    - 个人识别信息（PII）的保留期限是否符合隐私法规？
    - 是否建立了保留期限到期提醒机制？
    - 超过保留期限的数据是否按程序处理？
  - **安全销毁**：
    - 是否制定了数据销毁程序，确保数据不可恢复？
    - 电子数据是否使用符合标准的擦除或销毁方法？
    - 物理介质（硬盘、U盘、纸质文件）的销毁是否安全可控？
    - 数据销毁是否有记录和审计跟踪？
  - **访问终止**：
    - 项目结束或人员离职时，是否及时撤销数据访问权限？
    - 是否回收或禁用离职人员的账户、密钥和访问凭证？
    - 是否审查和清理离职人员访问或拥有的数据副本？

## B.3 分析方法检查清单

分析方法是将原始数据转化为有意义研究结论的关键环节。科学的分析方法选择和严谨的验证程序是确保研究结论可靠性的基础。本部分检查清单涵盖数据预处理、分析方法选择、结果验证以及软件与计算四个核心维度。

### B.3.1 数据预处理检查项

数据预处理是数据分析的基础工作，直接影响后续分析的质量和可靠性。

- [ ] **数据清洗是否已完成**
  - **错误值处理**：
    - 是否识别并处理了数据录入错误（如超出合理范围的值、不可能的日期）？
    - 是否使用领域知识和逻辑规则检测错误值？
    - 错误值的纠正是否基于可验证的信息来源？
    - 是否记录了所有错误值的识别和处理过程？
  - **不一致处理**：
    - 是否识别并解决了数据内部的不一致性（如矛盾的回答、不匹配的日期）？
    - 是否统一了不同数据源或时间点的格式和标准？
    - 是否处理了分类变量的拼写变体和同义词？
    - 是否建立了数据一致性规则和验证程序？
  - **重复记录处理**：
    - 是否识别了数据集中的重复记录？
    - 是否区分了真正的重复和合法的多重观测？
    - 重复记录的删除或合并是否基于明确的规则？
    - 是否保留了重复记录处理的审计跟踪？

- [ ] **缺失值处理策略是否已选择并实施**
  - **缺失模式分析**：
    - 是否分析了缺失数据的模式（完全随机缺失MCAR、随机缺失MAR、非随机缺失MNAR）？
    - 是否使用了Little's MCAR检验或其他方法评估缺失机制？
    - 是否分析了缺失数据与研究变量之间的关系？
  - **缺失值处理方法选择**：
    - **列表删除（完整案例分析）**：是否仅在MCAR假设成立且缺失比例较低时使用？
    - **成对删除**：是否意识到可能导致的样本量不一致和标准误偏差？
    - **均值/众数/中位数插补**：是否仅在探索性分析或缺失比例极低时使用？
    - **回归插补**：是否考虑了插补值缺乏变异性导致的方差低估？
    - **热卡插补/冷卡插补**：是否基于相似观测进行插补？
    - **多重插补（MI）**：是否使用Rubin的方法生成多个插补数据集并整合结果？
    - **最大似然估计（FIML）**：是否在结构方程模型等分析中使用FIML处理缺失数据？
  - **敏感性分析**：
    - 是否比较了不同缺失值处理方法对分析结果的影响？
    - 是否进行了假设性的MNAR分析以评估结果稳健性？

- [ ] **异常值检测与处理是否已进行**
  - **统计方法**：
    - 是否使用了基于分布的方法（如Z分数、IQR方法）识别异常值？
    - 是否使用了多元统计方法（如马氏距离、库克距离）识别多元异常值？
    - 是否使用了基于聚类或密度的方法（如DBSCAN、LOF）识别异常值？
  - **可视化方法**：
    - 是否使用箱线图、散点图、QQ图等可视化工具识别异常值？
    - 是否使用降维可视化（如PCA、t-SNE）检查多元异常值？
  - **领域知识**：
    - 是否结合领域知识判断统计异常值是否为真正的错误？
    - 是否区分了数据录入错误、测量误差和真实的极端观测？
  - **处理策略**：
    - 是否根据异常值的性质选择适当的处理方法（删除、修正、保留、Winsorize、转换）？
    - 是否进行了敏感性分析以评估异常值对结果的影响？

- [ ] **数据转换是否已进行**
  - **标准化**：
    - 是否根据分析需要选择了适当的标准化方法（Z标准化、Min-Max标准化、小数定标）？
    - 标准化参数（均值、标准差、最小/最大值）是否基于训练数据计算并应用于测试数据？
  - **归一化**：
    - 对于需要归一化的分析（如神经网络、距离计算），是否进行了适当的归一化处理？
  - **对数变换**：
    - 对于右偏分布的数据，是否考虑了对数变换（自然对数、常用对数）？
    - 对于含零或负值的数据，是否使用了ln(x+1)或其他适当的变换？
  - **类别编码**：
    - 是否根据分析需要选择了适当的编码方法（哑变量编码、独热编码、序数编码、目标编码）？
    - 是否处理了高基数类别变量的特殊问题？
  - **其他变换**：
    - 是否根据数据特征和分析需求使用了其他变换（平方根变换、倒数变换、Box-Cox变换、Yeo-Johnson变换）？
    - 变换是否改善了数据的正态性、线性关系或方差齐性？

- [ ] **数据重结构化是否已完成**
  - **宽格式与长格式转换**：
    - 是否根据分析需要（如重复测量分析、多层模型）进行了宽格式与长格式的转换？
    - 转换过程中是否保持了数据的完整性和标识符的正确关联？
  - **聚合与拆分**：
    - 是否根据分析单元（个体、群体、组织）对数据进行了适当的聚合或拆分？
    - 聚合操作是否考虑了数据的层次结构和依赖关系？
  - **合并与连接**：
    - 是否使用适当的合并方法（内连接、左连接、右连接、全连接）整合了多个数据源？
    - 合并操作是否保留了所有必要的记录和变量？
    - 是否处理了合并过程中的重复记录和缺失值？

### B.3.2 分析方法选择检查项

分析方法的选择直接影响研究结论的可靠性和有效性，必须与研究问题、数据特征和理论框架相匹配。

- [ ] **分析方法是否与研究问题和数据特征匹配**
  - 选择的分析方法是否能够有效回答研究问题？
  - 分析方法是否适合数据的测量层次（定类、定序、定距、定比）？
  - 分析方法是否考虑了数据的分布特征（正态性、方差齐性）？
  - 分析方法是否适合样本量的大小和结构？
  - 分析方法是否考虑了数据的依赖结构（独立观测、重复测量、层次结构）？

- [ ] **统计假设是否已明确**
  - **正态性假设**：
    - 对于参数检验，是否评估了变量的正态性分布（直方图、Q-Q图、Shapiro-Wilk检验、Kolmogorov-Smirnov检验）？
    - 当正态性假设不成立时，是否考虑了数据变换或非参数替代方法？
  - **独立性假设**：
    - 是否评估了观测之间的独立性？
    - 对于重复测量或层次数据，是否使用了适当的方法处理依赖性（如混合模型、GEE）？
  - **方差齐性假设**：
    - 对于方差分析、t检验等方法，是否评估了组间方差齐性（Levene检验、Bartlett检验）？

    - 结果是否对模型假设的变化具有稳健性？
    - 是否使用了模型平均或集成方法整合多个模型的结果？

- [ ] **多重比较校正是否已应用**
  - **族错误率控制**：
    - 是否识别了需要控制的假设族？
    - 是否使用了适当的校正方法控制族错误率（FWER）？
    - **Bonferroni校正**：是否对每个检验使用α/m的显著性水平？
    - **Holm-Bonferroni（逐步Bonferroni）**：是否使用逐步程序提高检验效能？
    - **Hochberg程序**：是否在适当条件下使用 Hochberg 程序提高效能？
  - **错误发现率控制**：
    - 在探索性分析或大规模检验中，是否使用FDR控制方法？
    - **Benjamini-Hochberg（BH）程序**：是否使用BH程序控制FDR？
    - **Benjamini-Yekutieli（BY）程序**：是否在存在依赖关系时使用BY程序？
    - **q值**：是否报告了q值（FDR控制下的p值等价物）？
  - **多重比较透明度**：
    - 是否报告了所有进行的检验，而不仅是显著的？
    - 是否报告了使用的校正方法和调整后的显著性水平？
    - 是否区分了预先设定的假设和探索性分析？

- [ ] **效应量是否已报告**
  - **效应量指标选择**：
    - 对于均值差异，是否报告了Cohen's d、Hedges' g或Glass's Δ？
    - 对于相关关系，是否报告了Pearson r、Spearman ρ或Kendall τ？
    - 对于分类变量关联，是否报告了Cramér's V、φ系数或优势比（OR）？
    - 对于方差分析，是否报告了η²、偏η²或ω²？
    - 对于回归模型，是否报告了R²、调整R²或 Cohen's f²？
  - **效应量解释**：
    - 是否提供了效应量大小的解释框架（如Cohen的传统标准：小=0.2，中=0.5，大=0.8）？
    - 是否考虑了领域特定的效应量解释标准？
    - 是否报告了未标准化效应量（原始单位）以增强可解释性？
  - **置信区间**：
    - 是否报告了效应量的置信区间？
    - 置信区间是否用于评估效应量估计的精确性和统计显著性？

- [ ] **置信区间是否已计算和报告**
  - **置信区间类型**：
    - 是否为关键统计量（均值、比例、相关系数、效应量）计算了置信区间？
    - 是否根据数据特征选择了适当的置信区间方法（正态近似、Bootstrap、精确方法）？
  - **Bootstrap置信区间**：
    - 在分布假设不满足或复杂统计量情况下，是否使用了Bootstrap方法？
    - 是否使用了适当的Bootstrap置信区间类型（百分位法、偏差校正加速法BCa）？
    - Bootstrap重复次数是否足够（通常≥1000）？
  - **置信区间解释**：
    - 是否正确解释了置信区间的含义（重复抽样下的区间覆盖概率）？
    - 是否避免了对置信区间的常见误解（如"95%置信区间包含真实参数的概率是95%"）？
    - 是否使用置信区间评估了估计的精确性和实际意义？

### B.3.4 软件与计算检查项

软件与计算工具的选择和使用直接影响分析的可重复性和可靠性。

- [ ] **分析工具选择是否适当**
  - **统计软件选择**：
    - 是否根据分析需求选择了适当的统计软件（R、Python、SPSS、Stata、SAS、Mplus、AMOS）？
    - 软件的功能是否满足研究所需的分析方法？
    - 软件的可及性（许可、成本）是否考虑？
  - **专业软件选择**：
    - 对于特定分析方法，是否选择了专业软件（NVivo、Atlas.ti用于定性分析；Gephi、UCINET用于网络分析）？
    - 专业软件的使用是否经过培训和验证？
  - **软件组合使用**：
    - 是否整合了多种软件工具以发挥各自优势？
    - 数据在不同软件间的转换是否保持了完整性和一致性？

- [ ] **代码可重复性是否已确保**
  - **文档化**：
    - 分析代码是否包含清晰的注释说明每个步骤的目的和逻辑？
    - 是否提供了README文件说明代码结构、运行环境和依赖关系？
    - 是否记录了所有分析决策的理由和依据？
  - **版本控制**：
    - 是否使用版本控制系统（如Git）管理代码？
    - 是否定期提交代码变更并编写清晰的提交信息？
    - 是否为重要版本打标签以便回溯？
  - **环境记录**：
    - 是否记录了分析环境的完整配置（操作系统、软件版本、软件包版本）？
    - 是否使用了虚拟环境（如conda、virtualenv）或容器（如Docker）隔离分析环境？
    - 是否提供了环境配置文件以便他人复现分析环境？

- [ ] **计算资源是否已评估和规划**
  - **内存需求**：
    - 是否评估了数据加载、处理和分析的内存需求？
    - 对于大数据集，是否使用了分块处理、内存映射或数据库解决方案？
    - 是否监控了内存使用情况并优化了内存使用效率？
  - **CPU需求**：
    - 是否评估了计算密集型任务（如蒙特卡洛模拟、自助法、复杂模型拟合）的CPU需求？
    - 是否使用了并行计算（多核、集群、云计算）加速计算？
    - 是否优化了算法效率以减少计算时间？
  - **GPU需求**：
    - 对于深度学习、大规模矩阵运算等任务，是否评估了GPU加速的需求？
    - 是否正确配置了GPU环境和库（如CUDA、cuDNN）？
    - 是否监控了GPU使用效率和内存占用？
  - **存储需求**：
    - 是否评估了原始数据、处理数据、分析结果和备份的存储需求？
    - 是否选择了适当的存储解决方案（本地存储、网络存储、云存储）？
    - 是否制定了存储容量规划和扩展策略？

- [ ] **软件包和库是否已记录**
  - **主要软件包**：
    - 是否列出了分析中使用的所有主要软件包和库？
    - 是否记录了每个软件包的名称、版本号和来源？
    - 是否说明了每个软件包在研究中的用途？
  - **R包记录示例**：
    - 是否使用`sessionInfo()`或`devtools::session_info()`记录R环境？
    - 是否使用`renv`或`packrat`创建可复现的R环境？
  - **Python包记录示例**：
    - 是否使用`pip freeze > requirements.txt`或`conda env export > environment.yml`记录Python环境？
    - 是否使用Poetry或Pipenv管理依赖关系？
  - **版本特异性**：
    - 是否说明了关键软件包的版本特异性（某些功能仅在特定版本中可用）？
    - 是否记录了软件包版本更新可能带来的影响？

- [ ] **计算错误和警告是否已检查和处理**
  - **收敛问题**：
    - 对于迭代算法（如最大似然估计、EM算法），是否检查了收敛性？
    - 是否设置了适当的收敛标准和最大迭代次数？
    - 对于未收敛的情况，是否尝试了不同的初始值或算法设置？
    - 是否报告了收敛诊断信息和最终收敛状态？
  - **奇异矩阵和多重共线性**：
    - 是否检查了矩阵奇异性和多重共线性问题？
    - 是否计算了方差膨胀因子（VIF）或条件数来诊断共线性？

  - **边界约束和非法值**：
    - 是否检查了参数估计是否超出了合理的边界（如概率超出[0,1]，方差为负）？

  - **警告信息处理**：
    - 是否仔细检查和理解了所有软件生成的警告信息？
    - 是否区分了可以安全忽略的警告和需要处理的警告？
    - 是否记录了所有警告信息及其处理方式？

  - **定期备份**：

    - 是否保留了历史版本的备份？
  - **归档存储**：

  - **可访问性和共享**：

    - 是否提供了适当的许可协议和使用说明？
